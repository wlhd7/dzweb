# Specification: "经典案例" (Case) 模块化重构

## 1. Overview
重构现有的“经典案例” (Case) 模块，使其具备高度的可扩展性和管理灵活性。管理员可以通过后台管理界面动态创建、编辑和删除案例模块。每个模块将作为一个独立的页面展示，支持按需添加文案和图片内容，并能通过多语言支持适应国际化需求。

## 2. Functional Requirements
### 2.1 模块化管理 (Admin Interface)
- **列表显示**：在“经典案例”主页的左侧栏展示所有已创建模块的标题。
- **新增入口**：管理员登录后，左侧栏右上角显示“新增”按钮。
- **模块创建/编辑面板**：
  - **标题 (必需)**：支持中、英、日三种语言输入。
  - **Slug (必需)**：用户自定义 URL 片段（例如 `robot-welding`），用于生成独立页面链接。
  - **内容流管理**：通过带有“+”号的方框交互界面添加新内容。
    - 点击“+”号弹出选择框：**添加文案** 或 **添加图片**。
    - **添加文案**：文本输入框，支持 I18n（中英日）。
    - **添加图片**：文件上传控件，遵循现有的 WebP 转换和缩略图生成规范。
    - 每个新内容项下方自动显示一个新的“+”方框，用于继续添加内容。
  - **排序与删除**：
    - 每个内容项（文案或图片）提供“上移”、“下移”按钮，实现按钮式逻辑排序。
    - 每个内容项支持单独删除。
- **模块删除**：管理员可删除整个案例模块。

### 2.2 独立页面展示 (User Interface)
- **动态路由**：根据自定义 Slug 生成独立页面（例如 `/case/robot-welding`）。
- **页面布局**：
  - 左侧栏：持续显示所有案例模块的标题列表，方便快速切换。
  - 主内容区：按照管理员设置的顺序顺序渲染文案（段落）和图片（支持响应式 WebP 加载）。
- **面包屑导航**：支持从“首页 > 经典案例 > [案例标题]”的层级导航。

### 2.3 多语言支持 (I18n)
- 模块标题、Slug 以及文案内容必须支持中、英、日三语翻译。
- 遵循项目现有的 Flask-Babel 国际化标准。

### 2.4 资源完整性协议
- **同步删除**：删除案例模块或模块内的单张图片时，必须物理删除 `static/uploads/` 下对应的图片文件。
- **同步更新**：更换图片时，自动清理旧的物理文件。

## 3. Non-Functional Requirements
- **性能优化**：所有上传的案例图片需自动转换为 WebP 格式，并生成 400x300 的缩略图供预览。
- **安全性**：案例管理接口需受现有的管理员密码保护 (`ADMIN_PASSWORD`)。
- **SEO**：自定义 Slug 结合三语标题，优化案例页面的搜索引擎收录。

## 4. Acceptance Criteria
- [ ] 管理员能成功创建带有 Slug 和三语标题的新案例模块。
- [ ] 能在案例内动态添加、删除文案和图片，并能通过按钮调整顺序。
- [ ] 案例图片上传后自动生成 WebP 格式及缩略图。
- [ ] 通过自定义 Slug 能正确访问对应的案例独立页面。
- [ ] 删除案例或图片后，物理文件从磁盘移除。
- [ ] 所有静态文本和动态内容在切换语言时正确显示。

## 5. Out of Scope
- 管理员后台的实时预览功能。
- 案例模块的分组或分类管理（目前统一在侧边栏扁平化显示）。
