# Specification - nav_cleanup_20250228

## 概览
**目标**：优化用户导航体验并清理已弃用的功能。
**背景**：目前用户登录后缺乏快速回到个人主页的入口。同时，“设置用户名颜色”和“周末加班单”功能已不再需要，需进行彻底清理。

## 功能需求
- **导航栏增强**：
  - 在全局页面右上角的认证状态区域，在“用户名”和“注销”链接之间插入一个新的链接：“主页”。
  - 该链接应以纯文本形式展示，点击后跳转至用户的个人控制面板 (`user.userhome`)。
- **应用清理**：
  - **设置用户名颜色**：
    - 从数据库 `apps` 表中删除对应的记录。
    - 移除后端路由 `user.set_color`。
    - 删除关联的模板文件 `dzweb/templates/user/set-color.html` 和任何专属的后端逻辑。
  - **周末加班单**：
    - 从数据库 `apps` 表中删除对应的记录。
    - 移除后端路由 `user.weekend_overtime`。
    - 删除关联的模板文件 `dzweb/templates/user/weekend-overtime.html`。
    - 移除后端 `user.py` 中与之相关的逻辑（包括 `cleanup_old_data`）。

## 验收标准
- [ ] 登录用户可以在页面右上角看到“主页”链接，并能正确跳转至个人控制面板。
- [ ] 个人主页的侧边栏应用列表中不再出现“设置用户名颜色”和“周末加班单”。
- [ ] 直接尝试访问 `/user/set-color` 或 `/user/weekend-overtime` 应返回 404 错误。
- [ ] 数据库 `apps` 表中不再包含上述两项的记录。

## 超出范围 (Out of Scope)
- 数据库中 `staffs` 表的列清理。
- 导航栏整体样式的重构。
