# Specification - product_ui_optimization_20250228

## 概览
**目标**：优化产品创建/编辑页面的表单布局，使其更符合现代 UI 审美，并为管理员提供更便捷的导航入口。
**背景**：目前产品表单布局较为松散，且管理员在浏览产品时缺乏快速跳转到创建页面的直达链接。

## 功能需求
- **表单布局精细化 (`product/create` & `product/update`)**：
  - **标签对齐**：所有 `label` 采用固定宽度对齐，但显著缩短其与对应的 `input`/`select`/`textarea` 之间的间距。
  - **描述对齐**：“产品描述”标签需精确定位于 `textarea` 的左侧，且两者的顶部边沿处于同一水平高度。
  - **容器居中**：整个 `dblock-content` 表单容器在页面中水平居中显示。
  - **按钮美化**：“保存”按钮长度缩短为自适应文字长度（fit-content），不再占据整行或过长。
- **导航增强**：
  - **管理员快捷入口**：当管理员（admin）登录后，在“产品类别”侧边栏标题的右侧增加一个文字链接“新增”。
  - **跳转目标**：点击该“新增”链接后，导航至 `product/create` 路由。

## 验收标准
- [ ] 产品表单各项（Label + Input）之间的水平间距明显减小。
- [ ] “产品描述”标签与 `textarea` 顶部对齐且并排显示。
- [ ] 表单整体在页面中央展示。
- [ ] “保存”按钮宽度仅包裹“保存”二字。
- [ ] 登录管理员账号后，侧边栏标题旁可见“新增”链接并能正确跳转。

## 超出范围 (Out of Scope)
- 响应式设计的重度重构。
- 后端保存逻辑的变更。
