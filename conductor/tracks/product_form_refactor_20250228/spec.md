# Specification - product_form_refactor_20250228

## 概览
**目标**：优化产品创建页面的交互逻辑与 UI 布局，提高管理员录入效率。
**背景**：目前产品分类与子类别之间缺乏联动，且表单布局较为拥挤，标签对齐不符合直觉。

## 功能需求
- **分类与子类别联动**：
  - 在 `product/create` 页面，当“产品分类”发生变化时，自动更新“子类别”下拉框。
  - **实现方式**：采用后端驱动逻辑。前端通过 API 接口获取当前选中分类对应的子类别列表。
  - **异常处理**：若该分类下没有子类别，则禁用子类别选择框，并清除现有选项。
- **UI 布局优化**：
  - **标签位置**：将“产品描述”标题移动到 `textarea` 输入框的左上方，采用块级显示。
  - **条目间距**：表单内的每个输入条目（Label + Input）之间增加明显的上下间距，建议使用 flex/grid 的 `gap` 属性或统一的 `margin-bottom`。

## 非功能需求
- **交互性能**：分类切换后的子类别更新应当无感且迅速。
- **代码整洁**：前端 JS 逻辑应模块化，避免在 HTML 中直接写大量内联脚本。

## 验收标准
- [ ] 切换“产品分类”后，“子类别”下拉框的内容同步更新。
- [ ] 无子类别的分类会导致子类别框变为禁用状态。
- [ ] “产品描述”标签准确位于文本域左上方。
- [ ] 表单各项之间有明显的垂直间距，视觉上不再拥挤。
