# Specification - admin_panel_refactor_20250228

## 概览
**目标**：将现有的“用户系统”转变为简化的“管理员后台（Admin Panel）”，移除冗余的注册逻辑和前端入口，通过单一预设密码进行认证。

## 功能需求
- **入口调整**：
  - 取消首页（`home.index`）上的“员工入口”链接。
- **认证逻辑重构**：
  - **移除注册功能**：彻底移除 `/auth/register` 路由及相关的 `register.html` 模板。
  - **简化登录**：`/auth/login` 页面改为仅需输入“管理员密码”。移除用户名输入框。
  - **密码校验**：认证逻辑改为对比环境变量中预设的 `ADMIN_PASSWORD`。
  - **Session 管理**：登录成功后，Session 标记为管理员状态（不再依赖具体的数据库用户 ID）。
- **路由与 UI 更新**：
  - **User Panel -> Admin Panel**：所有的“个人主页”字样更新为“管理后台”。
  - **路由重命名**：
    - `/user/edit-produt-permisssion` -> `/user/edit-product`。
    - `/user/edit-hir-permission` -> `/user/edit-hire`。
  - **标题更新**：
    - “编辑产品信息权限确定” -> “编辑产品信息”。
    - “编辑招聘信息权限确定” -> “编辑招聘信息”。
- **权限范围**：
  - 管理员拥有对所有注册应用（`apps` 表中的条目）的全局访问权。

## 非功能需求
- **安全性**：管理员密码必须通过环境变量注入，不得提交到代码库。
- **一致性**：确保导航栏中的“主页”链接指向新的管理员主页。

## 验收标准
- [ ] 首页不再显示“员工入口”。
- [ ] 访问 `/auth/register` 应当返回 404。
- [ ] 登录页面仅显示密码输入框，输入正确的环境变量密码即可进入 `/user/`。
- [ ] 侧边栏及页面标题已按新规范更新。
- [ ] 路由 `/user/edit-product` 和 `/user/edit-hire` 功能正常。

## 超出范围 (Out of Scope)
- 数据库 `users` 表的物理删除（暂定逻辑层面停用）。
- 多管理员支持。
