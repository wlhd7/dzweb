# Specification - remove_app_password_20250228

## 概览
**目标**：简化用户在个人主页访问应用的过程。
**背景**：目前用户需要通过口令手动添加应用到个人主页。为了提高用户体验，我们将移除口令验证逻辑，并在用户主页的左侧侧边栏中直接展示所有可用的应用链接。

## 功能需求
- **移除口令验证**：彻底重构后端逻辑，移除现有的“通过口令添加应用”的功能和对应的 API 接口。
- **所有应用展示**：在用户登录后的个人主页（User Home），侧边栏将自动列出系统中定义的所有可用应用。
- **点击直达**：点击应用链接后，用户应能直接跳转到对应的应用 URL，不再进行任何口令或权限的二次拦截。
- **UI 调整**：将应用列表（App List）集成到现有的个人主页侧边栏导航中。

## 非功能需求
- **性能**：应用列表的加载不应显著延迟个人主页的渲染速度。
- **安全性**：由于移除了口令验证，需确保应用本身的 URL 安全性（这可能超出了本 Track 的范围，但应予以关注）。

## 验收标准
- [ ] 用户登录后，左侧侧边栏中可见所有应用的名称和链接。
- [ ] 点击任何应用链接都能成功跳转，且不再弹出“输入口令”的对话框或提示。
- [ ] 后端代码中不再保留用于“口令添加应用”的路由和验证逻辑。

## 超出范围 (Out of Scope)
- 针对单个应用的精细化权限管理（本 Track 目标是显示“所有应用”）。
- 应用自身的业务逻辑开发。
