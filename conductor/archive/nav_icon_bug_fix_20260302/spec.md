# Specification: Home UI Navigation Icon Bug Fix

## Overview
修复首页 UI 中的一个图标渲染逻辑 Bug。当用户点击“关于我们”栏目内的链接时，“产品类别”侧边栏的大类图标会被错误地渲染为子类别图标样式。

## Functional Requirements
1.  **图标状态保持 (Icon Persistence)**:
    - 确保在点击非产品类链接（如“关于我们”系列链接）时，“产品类别”栏的顶级图标保持其默认状态（通常为大类圆点图标）。
    - 修复可能存在的 CSS 类名冲突或 JS 逻辑判断错误，防止全局状态错误地影响侧边栏图标。
2.  **路由感应优化 (Route Awareness)**:
    - 检查侧边栏图标切换逻辑（可能是基于 `request.path` 或 `active` 类名）。
    - 仅当真正进入产品子类别页面时，才触发子类别图标显示。

## Non-Functional Requirements
- **UI 一致性**: 修复后的表现必须符合工业严谨的视觉风格。
- **性能**: 不应引入显著的 JS 执行负担。

## Acceptance Criteria
- [ ] 点击“公司简介”、“发展历程”等关于我们的链接后，产品类别栏的大类图标保持不变。
- [ ] 正常切换产品类别时，子类别的图标显示逻辑依然正常（未被破坏）。
- [ ] 页面跳转后，图标状态能正确持久化显示。

## Out of Scope
- 修改“关于我们”页面的具体内容。
- 重构整个侧边栏的布局结构。
