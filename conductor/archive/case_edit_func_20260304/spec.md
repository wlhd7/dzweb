# Track Specification: Classic Case Edit Functionality

## Overview
此 Track 将在管理员后台（Admin Panel）实现“经典案例”模块的编辑功能。管理员能够更新现有案例的标题、正文内容以及图片，并确保数据库记录与服务器物理文件的一致性。

## Functional Requirements
- **权限控制**：编辑功能仅限已登录的管理员在 `/admin` 路径下访问。参考产品编辑（Product Edit）的权限控制逻辑（`@login_required`）。
- **模块标题编辑**：在案例详情页标题旁增加“编辑”按钮，支持修改 `case_modules` 表中的 `title_zh`（中文标题）。同时，系统应根据新标题自动重新生成 `slug` 并更新数据库，以保持 URL 的语义一致性。
- **文案内容编辑**：为每个文案项（`case_contents` 且 `type='text'`）增加“编辑”按钮，支持修改 `content_zh`（中文正文）。
- **图片更新与维护**：
  - 为每个图片项（`case_contents` 且 `type='image'`）支持上传新图片替换原有图片。
  - **资源完整性**：上传新图后，必须自动物理删除旧的原始图片、缩略图及其对应的 WebP 版本，防止存储空间膨胀（遵循 `GEMINI.md` 规范）。
- **多语言处理**：目前仅暴露并支持中文（`_zh`）内容的编辑，英、日语字段暂不处理（在后端保留扩展性，但不强制同步更新）。

## Non-Functional Requirements
- **数据一致性**：确保数据库更新与物理文件操作的原子性。
- **安全性**：防止未经授权的访问和非法表单提交。
- **交互一致性**：使用 Modal（弹窗）形式进行“原地”编辑，保持与现有案例管理界面一致。

## Acceptance Criteria
- [ ] 管理员在后台点击“编辑”后，能加载对应案例/内容的当前中文内容。
- [ ] 修改文字内容并提交后，数据库对应记录被正确更新。
- [ ] 上传新图提交后，旧图文件（含缩略图和 WebP）从服务器物理删除，新图显示正常。
- [ ] 前端页面（中文版）能立即反映更新后的内容。
- [ ] 非管理员尝试访问编辑路径时会被拦截。

## Out of Scope
- 英文（`_en`）和日文（`_ja`）内容的编辑界面。
- 案例内容的重新排序（Sort Order 调整）—— 现有功能已支持。
- 批量编辑功能。
